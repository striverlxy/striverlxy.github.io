redis击杀之路01---字符串

SET

语法

SET key value [EX second] [PX millisecond] [NX|XX]

功能

将字符串值value关联到key上
如果key已经被关联有其他值，SET之后着会以新值覆盖旧值，并且无视类型
当SET针对一个带有^生存时间的key进行设置的时候，该key原有的生存时间将被清除

可选参数

  • EX second 将键的过期时间设置成second秒
    • SET key value Ex second <==> SETEX key second value
  • PX millisecond 将键的过期时间设置成millisecond毫秒
    • SET key value Px millisecond <==> PSETEX key millisecond value
  • NX 只在键不存在的时候才能设置成功
    • SET key value NX <==> SETNX key value
  • XX 只在键存在的时候才能操作

代码演示

对不存在的键【key】设置

127.0.0.1:6379> exists key
(integer) 0
127.0.0.1:6379> set key "value"
OK
127.0.0.1:6379> get key
"value"

对已存在的键【key】设置

127.0.0.1:6379> exists key
(integer) 1
127.0.0.1:6379> get key
"value"
127.0.0.1:6379> set key "new-value"
OK
127.0.0.1:6379> get key
"new-value"

EX设置过期时间(秒级)

127.0.0.1:6379> set key-with-expire-seconds-time "value2" EX 100
OK
127.0.0.1:6379> get key-with-expire-seconds-time
"value2"
127.0.0.1:6379> TTL key-with-expire-seconds-time
(integer) 83
127.0.0.1:6379> setex key-with-expire-seconds-time 100 "value3"
OK
127.0.0.1:6379> get key-with-expire-seconds-time
"value3"
127.0.0.1:6379> TTL key-with-expire-seconds-time
(integer) 76

PX设置过期时间(毫秒级)

127.0.0.1:6379> set key-with=expire-millisecond-time "value4" PX 50000
OK
127.0.0.1:6379> get key-with=expire-millisecond-time
"value4"
127.0.0.1:6379> PTTL key-with=expire-millisecond-time
(integer) 39598
127.0.0.1:6379> psetex key-with=expire-millisecond-time 50000 "value4"
OK
127.0.0.1:6379> get key-with=expire-millisecond-time
"value4"
127.0.0.1:6379> PTTL key-with=expire-millisecond-time
(integer) 41905

使用NX设置

127.0.0.1:6379> exists unexists
(integer) 0
127.0.0.1:6379> set unexists "not exist" NX
OK
127.0.0.1:6379> get unexists
"not exist"
127.0.0.1:6379> set existKey "exist key"
OK
127.0.0.1:6379> exists existKey
(integer) 1
127.0.0.1:6379> set existKey "set exist key again" NX
(nil)
127.0.0.1:6379> get existKey
"exist key"
127.0.0.1:6379> exists unexists
(integer) 0
127.0.0.1:6379> setnx unexists "not exist"
OK
127.0.0.1:6379> get unexists
"not exist"
127.0.0.1:6379> set existKey "exist key"
OK
127.0.0.1:6379> exists existKey
(integer) 1
127.0.0.1:6379> setnx existKey "set exist key again"
(nil)
127.0.0.1:6379> get existKey
"exist key"

使用XX设置

127.0.0.1:6379> exists unexists
(integer) 0
127.0.0.1:6379> set unexists "unexist key" XX
(nil)
127.0.0.1:6379> get unexists
(nil)
127.0.0.1:6379> set existKey "exist key"
OK
127.0.0.1:6379> exists existKey
(integer) 1
127.0.0.1:6379> set existKey "set exist key again" XX
OK
127.0.0.1:6379> get existKey
"set exist key again"

GET

语法

get key

功能

获取键的值

返回值

如果键不存在，则返回nil,否则返回键的值
如果获取到的键的值不是字符串类型，则会返回错误，get只会获取字符串类型的键值

代码演示

对不存在的键进行获取

127.0.0.1:6379> exists unexistKey
(integer) 0
127.0.0.1:6379> get unexistKey
(nil)

对已存在的键进行获取

127.0.0.1:6379> exists existKey
(integer) 1
127.0.0.1:6379> get existKey
"value"

对类型非字符串的键进行获取

127.0.0.1:6379> LPUSH db value1 value2
(integer) 2
127.0.0.1:6379> GET db
(error) WRONGTYPE Operation against a key holding the wrong kind of value

GETSET

语法

GETSET key value

功能

将键的值设置成value,并返回设置前的旧值

返回值

返回给定键的旧值
如果键不存在，则返回nil
当键存在，但是类型不是字符串的时候，则会返回一个错误

代码演示

键存在

127.0.0.1:6379> set key "value111"
OK
127.0.0.1:6379> getset key "value222"
"value111"

键不存在

127.0.0.1:6379> exists key
(integer) 0
127.0.0.1:6379> getset key "value111"
(nil)

键值类型不是字符串

127.0.0.1:6379> LPUSH db value1 value2
(integer) 2
127.0.0.1:6379> getset db
(error) ERR wrong number of arguments for 'getset' command

STRLEN

语法

strlen key

功能

返回键存储字符串值的长度

返回值

当键存在时，返回其键存储的字符串长度
当键不存在是，返回0
当获取到键的值不是字符串类型，则保错

代码演示

键值存在

127.0.0.1:6379> set len "12345"
OK
127.0.0.1:6379> exists len
(integer) 1
127.0.0.1:6379> strlen len
(integer) 5

键不存在

127.0.0.1:6379> exists len2
(integer) 0
127.0.0.1:6379> strlen len2
(integer) 0

键存储的不是字符串

127.0.0.1:6379>  LPUSH db value1 value2
(integer) 2
127.0.0.1:6379> strlen db
(error) WRONGTYPE Operation against a key holding the wrong kind of value

APPEND

语法

APPEND key value

功能

将值设置到键保存的值之后，如果键不存在，则初始化key，并将value设置进去，相当于 set key value, 如果键值不是字符串类型，则返回错误

返回值

追加value之后，值的总长度

代码演示

key不存在

127.0.0.1:6379> exists key3
(integer) 0
127.0.0.1:6379> append key3 "123"
(integer) 3
127.0.0.1:6379> get key3
"123"

key存在

127.0.0.1:6379> exists key3
(integer) 1
127.0.0.1:6379> append key3 "456"
(integer) 6
127.0.0.1:6379> get key3
"123456"

key的值类型不是字符串

127.0.0.1:6379> lpush key4 "11" "22" "33"
(integer) 3
127.0.0.1:6379> append key4 "44"
(error) WRONGTYPE Operation against a key holding the wrong kind of value

SETRANGE

语法

SETRANGE key offset value

功能

从偏移量offset开始，用value覆盖原key存储的value值
不存在的key则当作空字符串处理
SETRANGE会保证要被设置的key值有足够的长度够偏移量来设置新值，如果原字符串长度小于偏移量(比如原串长度为3，偏移量为4), 这种情况下，命令会自动在原串后用零字节(“\x00”)填充
因为redis限制字符串大小最大只有512mb， 所以offset最大值为2^29-1(536870911)，如果超过这个值，将无法设置

返回值

返回设置之后字符串的总长度

代码演示

对空字符串的设置

127.0.0.1:6379> exists unexistKey
(integer) 0
127.0.0.1:6379> setrange unexistKey 2 "111"
(integer) 5
127.0.0.1:6379> get unexistKey
"\x00\x00111"

对非空字符串的设置

127.0.0.1:6379> set existKey "abcde"
OK
127.0.0.1:6379> setrange existKey 1 "f"
(integer) 5
127.0.0.1:6379> get existKey
"afcde"

GETRANGE

语法

GETRANGE key start end

功能

获取key值指定区域的值,[start, end]
负数的偏移量是从末尾开始计数的，比如-1为最后一位， -2为倒数第二位
GETRANGE保证区间不超过key值的最大值，超过的会被自动忽略

返回值

返回key值指定区域的值

代码演示

127.0.0.1:6379> exists existKey
(integer) 0
127.0.0.1:6379> getrange existKey 1 2
""
127.0.0.1:6379> set existKey "12345"
OK
127.0.0.1:6379> getrange existKey 1 2
"23"
127.0.0.1:6379> getrange existKey -1 -2  # 不允许逆向截取
""
127.0.0.1:6379> getrange existKey -2 -1
"45"
127.0.0.1:6379> getrange existKey 1 10086
"2345"

INCR

语法

INCR key

功能

为key存储的数字加1
如果key不存在，则先将key值初始化为0，再执行INCR操作
如果key存储的值不能转化为数字，则会返回一个错误
本操作限制在64位(bit)有符号数字范围内

返回值

返回加1操作之后的值

代码演示

key不存在

127.0.0.1:6379> exists existkey
(integer) 0
127.0.0.1:6379> incr existKey
(integer) 1
127.0.0.1:6379> get existKey
"1"

key存在

127.0.0.1:6379> set existKey "1"
OK
127.0.0.1:6379> exists existkey
(integer) 1
127.0.0.1:6379> incr existKey
(integer) 2
127.0.0.1:6379> get existKey
"2"

key存储的字符串无法转化为数字

127.0.0.1:6379> set existKey "abc"
OK
127.0.0.1:6379> incr existKey
(error) ERR value is not an integer or out of range

INCRBY

语法

INCRBY key icrement

功能

为key存储的数字加上增量incrment
key不存在时，则先初始化出key的值为0，在执行INCRBY操作
如果key存储的值不能转化为数字，则会返回一个错误
本操作限制在64位(bit)有符号数字范围内

返回值

在加上增量increment后，返回最新的key值

代码演示

key不存在

127.0.0.1:6379> exists existKey
(integer) 0
127.0.0.1:6379> incrby existKey 100
(integer) 100
127.0.0.1:6379> get existKey
"100"

key存在

127.0.0.1:6379> exists existKey
(integer) 1
127.0.0.1:6379> get existKey
"100"
127.0.0.1:6379> incrby existKey 100
(integer) 200
127.0.0.1:6379> get existKey
"200"

INCRBYFLOAT

语法

INCRBYFLOAT key increment

功能

对key值加上浮点数增量increment
key不存在，则先将key值初始化为0,再进行操作
key值或者increment都可以使用指数形式(1.0e2 ==> 100)
该操作最多保留小数点后17位
当key值不能被解析成双精度浮点数时，会返回一个错误

返回值

加上增量increment后，返回最新key的值

代码示范

key不存在

127.0.0.1:6379> exists decimal
(integer) 0
127.0.0.1:6379> incrbyfloat decimal "2.0"
"2"
127.0.0.1:6379> get decimal
"2"

key存在

127.0.0.1:6379> get decimal
"2"
127.0.0.1:6379> incrbyfloat decimal "10.5"
"12.5"
127.0.0.1:6379> get decimal
"12.5"

DECR

语法

DECR key

功能

为key存储的数字减1
key不存在时，则先初始化出key的值为0，在执行DECR操作
如果key存储的值不能转化为数字，则会返回一个错误
本操作限制在64位(bit)有符号数字范围内

返回值

减1之后，返回最新的key的值

代码演示

127.0.0.1:6379> exists existKey
(integer) 0
127.0.0.1:6379> decr existKey
(integer) -1
127.0.0.1:6379> get existKey
"-1"
127.0.0.1:6379> exists existKey
(integer) 1
127.0.0.1:6379> decr existKey
(integer) -2
127.0.0.1:6379> get existKey
"-2"

DECRBY

语法

DECRBY key decrement

功能

为key存储的数字减去减量decrement
key不存在时，则先初始化出key的值为0，在执行DECRBY操作
如果key存储的值不能转化为数字，则会返回一个错误
本操作限制在64位(bit)有符号数字范围内

返回值

返回键在执行减法操作之后的值

代码演示

127.0.0.1:6379> exists existKey
(integer) 0
127.0.0.1:6379> decrby existKey 100
(integer) -100
127.0.0.1:6379> get existKey
"-100"
127.0.0.1:6379> decrby existKey 48
(integer) -148
127.0.0.1:6379> get existKey
"-148"

MSET

语法

MSET key value [key value …]

功能

同时为多个键设置值
如果其中的某些键已存在，则会覆盖原有的值
MSET是个原子操作，所以不会出现一些值设置成功了，一些是设置失败的情况

返回值

总是返回OK

代码演示

127.0.0.1:6379> mset key1 "value1" key2 "value2"
OK
127.0.0.1:6379> mget key1 key2
1) "value1"
2) "value2"
127.0.0.1:6379> mset key1 "value11" key3 "value3"
OK
127.0.0.1:6379> mget key1 key2 key3
1) "value11"
2) "value2"
3) "value3"

MSETNX

语法

MSETNX key value [key value …]

功能

当且仅当所有key不存在时，同时为多个键设置值
如果有一个key是存在的，则当前操作失败
MSETNX为原子操作，所以要么全部成功，要么全部失败

返回值

设置成功，返回1
设置失败，返回0

代码演示

127.0.0.1:6379> msetnx key11 "value11" key22 "value22"
(integer) 1
127.0.0.1:6379> mget key11 key22
1) "value11"
2) "value22"
127.0.0.1:6379> exists key11
(integer) 1
127.0.0.1:6379> msetnx key11 "value99" key4 "value77"
(integer) 0
127.0.0.1:6379> mget key11 key4
1) "value11"
2) (nil)

MGET

语法

MGET key [key …]

功能

返回给定的一个或多个字符串键的值
如果给定的字符串键里面， 有某个键不存在， 那么这个键的值将以特殊值 nil 表示

返回值

MGET 命令将返回一个列表， 列表中包含了所有给定键的值

代码演示

127.0.0.1:6379> set key101 "101"
OK
127.0.0.1:6379> set key102 "102"
OK
127.0.0.1:6379> mget key101 key102 key103
1) "101"
2) "102"
3) (nil)
